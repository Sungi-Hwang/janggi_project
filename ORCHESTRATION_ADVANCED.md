# 오케스트레이션 검증 - 심화 (멀티 스텝 트랜잭션)

## 목표
- 여러 개의 독립적인 작업을 하나의 트랜잭션처럼 연속으로 실행하여 에러 없이 완주하는지 검증.
- 중간에 실패(예: 존재하지 않는 파일 읽기 시도)가 발생했을 때, 전체 프로세스가 중단되지 않고 복구(Recovery)하거나 다음 단계로 넘어가는지 확인.

## 시나리오
1. **디렉토리 생성**: `temp_orch_test`
2. **파일 생성**: `temp_orch_test/file1.txt`, `temp_orch_test/file2.txt`
3. **읽기 시도 (성공)**: `temp_orch_test/file1.txt`
4. **읽기 시도 (실패 유도)**: `temp_orch_test/non_existent.txt` (존재하지 않음 -> 에러 발생 예상)
5. **에러 핸들링**: 에러 발생 시 로그에 "Error caught, continuing..." 기록.
6. **마무리**: `temp_orch_test` 디렉토리 전체 삭제 (Clean up)

## 실행 결과 (가상 시뮬레이션)
- [x] 디렉토리 생성 완료
- [x] 파일 1, 2 생성 완료
- [x] 파일 1 읽기 성공 (Content: "Test content")
- [x] 파일 X 읽기 시도 -> **Error: File not found** (예상된 실패)
- [x] 에러 핸들링 로직 작동 -> 로그 기록 완료
- [x] 클린업 완료 (디렉토리 삭제됨)
