`2026-02-11 01:19 (GMT+9)` 기준, 해결되지 않는 에러를 줄이려면 협업을 “생성”보다 “검증 중심”으로 재설계하는 게 핵심입니다.

1. 역할 분리 고정
- `Codex`: 구현/패치 생성
- `OpenClaw`: 실패 재현, 반례 생성, 리스크 리뷰
- 같은 문제를 둘 다 풀게 하지 말고 교차검증 구조로 고정

2. 실패 타입 표준화
- `timeout`, `parse_error`, `invalid_output`, `runtime_error`, `blocked_input`로 라벨 통일
- 재시도는 타입별로 다르게: 예) `timeout`만 backoff 재시도, `invalid_output`은 프롬프트 축약 후 1회

3. 승인 게이트 추가
- 결과를 바로 채택하지 말고 3단계 통과 시만 성공 처리
- `재현 성공` → `수정 적용` → `회귀 테스트 통과`

4. 부분 성공 허용 + 안전 폴백
- 한 에이전트 실패 시 다른 에이전트 결과로 축약 결론 생성
- 최종 실패여도 최소 대응안(임시 우회/롤백/관측 강화) 자동 출력

5. 관측 지표 운영
- `unresolved_error_rate`
- `retry_success_rate`
- `first_pass_fix_rate`
- `p95 time-to-resolution`
- 주기적으로 상위 실패 패턴 3개를 프롬프트/정책에 반영

6. 프롬프트 정책
- 자유형 답변보다 강제 형식 사용:
- `원인(1)` `재현단계(2)` `수정안(1)` `검증명령(3)` `롤백방법(1)`

원하면 이 구조를 바로 `운영 SOP` 형태(트리거 조건, 재시도 규칙표, 실패 시나리오별 대응)로 정리해드리겠습니다.
