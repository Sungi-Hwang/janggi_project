주제: 디스코드 기반 다중 AI 토론 운영정책
역할: Con(비판/리뷰 관점).
아래 Pro 의견을 검토해서 답하라.
[Pro 의견]
`2026-02-11 01:32 (GMT+9)` 기준 디스코드 기반 다중 AI 토론 운영정책(실무형)입니다.

1. 역할 분리
- `Pro Agent`: 실행안 제시(근거 3, 액션 3)
- `Con Agent`: 리스크/반례 제시(위험 3, 완화 3)
- `Synth Agent`: 최종 결론 통합(결론/찬성근거/반대근거/즉시액션)

2. 출력 표준
- 모든 에이전트는 고정 템플릿 사용
- 필수 섹션 누락 시 `quality_fail` 처리
- 길이 제한(예: 12줄)으로 지연과 잡담 방지

3. 실패 코드 표준
- `timeout`, `input_invalid`, `tool_error`, `quality_fail`, `unknown`
- 모든 런에서 `error_code + evidence_path` 기록 필수

4. 재시도/폴백
- 재시도 최대 2회
- backoff+jitter 적용
- 합성 실패 시 `retry -> local_fallback`으로 종료 보장

5. Discord 채널 정책
- 운영 채널: 사용자용 결과만 출력
- 로그 채널: 타임라인/에러코드/런ID만 출력
- 멘션 스팸 방지(실패 반복 시 묶음 알림)

6. 운영 지표(SLO)
- `timeout_rate`, `retry_success_rate`, `fallback_rate`, `p95_latency`
- 임계치 초과 3회 연속 시 자동 “운영자 점검 필요” 발행

7. 거버넌스
- 프롬프트/타임아웃 변경은 버전 관리
- 정책 변경 시 A/B 런으로 성능 비교 후 반영
- 민감 주제는 안전 필터 + 인간 승인 단계 추가

원하면 이 정책을 바로 `Discord 운영용 SOP 체크리스트` 형식으로 변환해드리겠습니다.

요구사항:
- 반론 또는 보완점 3개
- Pro 주장의 취약점 2개
- 안전 대안 2개
- 만약 Pro가 전반적으로 타당하면 '주요 반대 없음'을 명시하고 보완점 중심으로 작성
형식: 불릿 위주, 14줄 이내.
현재 실행 상태/로그/파일 경로 언급 금지.
