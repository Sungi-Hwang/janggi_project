## 🛠️ 기술적 해결 사례 (핵심 기억)

### 1. Stockfish DLL 안정성 확보 ✅
- **문제**: FFI를 통한 직접 호출 시 메모리 누수 및 Access Violation 크래시 발생.
- **해결**: **"프로세스 격리(Process Isolation)"** 도입. 
  - `single_game_extract.dart`를 통해 각 게임을 별도 Dart 프로세스로 실행.
  - 배치 스크립트(`extract_full.ps1`)로 안정적인 자동화 추출 성공.

### 2. 묘수 추출 알고리즘 고도화 ✅
- **문제**: 단순히 `bestmove`만 저장하여 연속된 수순(2~3수) 대응 불가 및 힌트 오작동.
- **해결**: **"반복적 엔진 시뮬레이션(Iterative PV Extraction)"** 도입.
  - 엔진이 Mate를 발견하면 가상 보드에서 직접 말을 움직여가며 실제 외통까지의 전체 수순(PV)을 추출하여 `puzzles.json`에 저장.

### 3. FEN 기물 매핑 오류 수정 ✅
- **내용**: Fairy-Stockfish Janggi variant에서는 Horse='n', Elephant='b'를 사용함을 확인하여 매핑 로직 수정. (기존 'h', 'e'에서 변경)

## ✨ 구현된 주요 기능

### 1. 묘수풀이 모드 (Puzzle Mode)
- **데이터**: 100개 기보 전수 조사하여 1/2/3수 외통 문제 22개 추출 완료.
- **로직**: 사용자가 정답을 두면 상대방(AI)이 **자동으로 대응 수**를 두는 기능 구현.
- **힌트**: 정답 수순과 100% 일치하는 화살표 가이드 제공.

### 2. 형세 판단 바 (Evaluation Bar)
- **내용**: 체스닷컴 스타일의 시각적 유리 지표 구현. (`EvaluationBar` 위젯)
- **연동**: 힌트 클릭 시 Stockfish 엔진의 centipawn 점수를 실시간 반영.

### 3. 설정 및 사운드 시스템
- **설정**: 사운드 ON/OFF, 음량 조절, 장기판/기물 스킨 변경, AI 생각 시간 조절.
- **사운드**: 고품질 착수음, 포획음, 장군/멍군(국악풍), 승리(환호성) 사운드 연동 완료.
